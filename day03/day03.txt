크기 (단위)
    1. px
        절대 크기 지정방식, 내가 정해준 크기로 무조건 지정

    2.rem
        문서 요소 최상위 태그 = html의 font-size의 배율
        1rem = 16px(html font-size default value = 16px)
        2rem = 32px

    3. em
        상위 요소(부모) font-size의 배율 (특정 조건에 의해 바뀔수 있다)
        1.5em = 부모 요소 fot-size의 1.5배

    4. %
        부모 요소 혹은 기준이 되는 요소와의 비율

    5. vw, vh
        100vw(사용자 해상도의 100% 길이)
        100vh(사용자 해상도의 100% 높이)

        *100vw : (100%)
            (스크롤의 길이는 포함하지 않음)

    6. calc
        계산기능(특정요소의 길이를 제하기 위해)
            ex)
                height : calc(100vh - 80px)
                * 띄어쓰기 꼭 지켜야함

    그리고 해당 주소에 웹 페이지를 그리기 위한 데이터를 요청
    WAS(서버 도우미) ex) 톰캣 => 데이터 요청, 데이터 응답 등을 처리
    다시 웹브라우저에 전송, 웹 브라우저는 받아온 데이터를 토대로 페이지 파싱

    <-- 파싱과정 -->
    웹 브라우저에 전달된 페이지를 브라우저 로더가 분석
    html, css, js분리
    html(DOM tree), CSS(CSS OM), JS(자바 스크립트 엔진)

    DOMtree, CSS OM이 합쳐지면서 랜더링트리(F12(개발자 도구))를 만들어줌
    랜더링트리를 토대로 CSS가 정해지는 레이아웃과정을 거침 -> 크기, 위치 계산
    JS로드 (평가와 실행 -> JS 실행 컨텍스트)
    페인트(사용자 화면에 그려주는 것)

    사용자의 행위에 따라 css나 html의 구조적 변경이 일어남
        이경우에는 레이아웃 과정 -> 페인트 과정을 다시 그리는 
        리페인트 및 리레이아웃 과정이 존재

CSS 배경
    background-color : 배경색

    background-image : 배경 이미지

    background-repeat : 배경의 바둑판 반복식 여부를 조정(옵션)

    background-position : 반복되지 않은 배경의 이미지의 상대 위치를 설정

    background-attachment : 위치가 설정된 배경 이미지를 고정
                            스크롤과 상관없이 화면에서 배경이 이동하지 않습니다

    background-size : 배경의 크기

        cover -> 페이지에 가득 차게(꽉 채움)
                종횡비에 따라 이미지가 잘릴 수 있음

        contain -> 원본 종횡비에 맞게 사진이 전체가 보이도록
                    사진의 크기에 따라 여백이 생김

    축약(전체)
    background : 경로, 반복여부, 위치, 속성 / 사이즈
        ex) background: url(이미지 경로) no-repeat fixed
